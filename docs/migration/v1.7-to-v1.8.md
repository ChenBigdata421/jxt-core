# go-admin-core v1.7 → v1.8 迁移指南

## 概述

go-admin-core v1.8.0 进行了重要的架构优化,主要目标是:

1. **简化包结构** - 将常用包从 `sdk/pkg/*` 提升到根目录
2. **优化命名** - 缩短过长的包名 (`observability` → `observe`)
3. **解决冲突** - 消除包名冲突 (`tools/gorm/logger` → `tools/gorm/gormlog`)
4. **向后兼容** - 提供完整的兼容层,新旧路径在 v1.8.x-v1.9.x 期间都可用

**重要**: 本次升级是**非破坏性的**,旧代码无需立即修改即可继续工作。

## 变更概览

### 包路径变更

| 旧路径 | 新路径 | 说明 |
|--------|--------|------|
| `sdk/pkg/captcha` | `captcha` | 验证码功能 |
| `sdk/pkg/jwtauth` | `jwtauth` | JWT认证 |
| `sdk/pkg/response` | `response` | HTTP响应封装 |
| `sdk/pkg/casbin` | `casbin` | 权限管理 |
| `observability/audit` | `observe/audit` | 审计日志 |
| `tools/gorm/logger` | `tools/gorm/gormlog` | GORM日志适配器 |

### 兼容性时间线

```
v1.8.0-beta (2025-10-24)
  ├─ 新旧路径都可用
  └─ 旧路径标记为 Deprecated
  
v1.8.0 (2025-11-07)
  ├─ 稳定版发布
  └─ 旧路径显示废弃警告
  
v1.9.x (2025-11-08 ~ 2026-04-30)
  ├─ 维护期
  └─ 建议迁移到新路径
  
v2.0.0 (2026-05-01)
  └─ 旧路径移除 ⚠️
```

## API 兼容性

### ✅ 完全兼容的 API

所有公共 API 保持 100% 兼容,包括:

**captcha**:
- `SetStore(s base64Captcha.Store)`
- `DriverStringFunc() (id, b64s, answer string, err error)`
- `DriverDigitFunc() (id, b64s, answer string, err error)`
- `Verify(id, code string, clear bool) bool`

**jwtauth**:
- `type MapClaims`
- `type GinJWTMiddleware`
- `New(m *GinJWTMiddleware) (*GinJWTMiddleware, error)`
- `ExtractClaims(c *gin.Context) MapClaims`
- `ExtractClaimsFromToken(token *jwt.Token) MapClaims`
- `GetToken(c *gin.Context) string`

**response**:
- `Error(c *gin.Context, code int, err error, msg string)`
- `OK(c *gin.Context, data interface{}, msg string)`
- `PageOK(c *gin.Context, result interface{}, count int, pageIndex int, pageSize int, msg string)`
- `Custum(c *gin.Context, data gin.H)`

**casbin**:
- `type Logger`
- `Setup(db *gorm.DB, tableName string) *casbin.SyncedEnforcer`

**observe/audit**:
- `type Log`
- `type Record`
- `type Stream`
- 所有原有函数和常量

**tools/gorm/gormlog**:
- `New(config logger.Config) logger.Interface`

### ⚠️ 不兼容的情况

**无** - 本次升级完全向后兼容!

## 常见问题

### Q1: 必须立即迁移吗?

**A**: 不需要。旧路径在 v1.8.x 和 v1.9.x 期间仍然可用,有6个月的过渡期。但建议尽早迁移以避免未来的问题。

### Q2: 迁移会破坏现有代码吗?

**A**: 不会。我们提供了完整的兼容层,旧代码无需任何修改即可继续工作。

### Q3: 如何知道哪些文件需要更新?

**A**: 使用自动迁移脚本,它会自动扫描并显示所有需要更新的文件。或者手动搜索:

```bash
grep -r "sdk/pkg/captcha\|sdk/pkg/jwtauth\|sdk/pkg/response\|sdk/pkg/casbin" --include="*.go" .
```

### Q4: 迁移后编译失败怎么办?

**A**: 
1. 检查是否正确更新了所有导入路径
2. 运行 `go mod tidy` 清理依赖
3. 检查 `go.mod` 中的版本是否正确
4. 查看错误信息,确认是否有拼写错误

### Q5: 可以只迁移部分包吗?

**A**: 可以。您可以逐个包迁移,或者混合使用新旧路径。但建议一次性全部迁移,保持代码一致性。

### Q6: 如何回滚?

**A**: 
- 如果使用自动脚本: 恢复备份目录中的文件
- 如果手动迁移: 使用 `git revert` 或 `git reset`
- 降级依赖: `go get github.com/go-admin-team/go-admin-core@v1.7.x`

### Q7: IDE 显示 Deprecated 警告正常吗?

**A**: 如果您还在使用旧路径,显示 Deprecated 警告是正常的。这是提醒您迁移到新路径。迁移后警告会消失。

### Q8: go-admin-pro 需要特殊处理吗?

**A**: 不需要。使用相同的迁移方法即可。建议先在 go-admin-core 上测试,然后再迁移 go-admin-pro。
