# go-admin-core v1.8.0 重构计划

**版本**: v1.8.0-beta  
**日期**: 2025-10-17  
**状态**: 🚧 执行中  
**重构类型**: 渐进式重构（提供兼容层）

---

## 📋 执行摘要

本次重构旨在优化 go-admin-core 的包结构，移除 `sdk/pkg` 反模式，同时保持向后兼容性。

### 核心目标

1. ✅ **消除 sdk/pkg 反模式** - 将常用包提升到根目录
2. ✅ **简化包名** - 缩短过长的包名
3. ✅ **解决命名冲突** - 避免 import 歧义
4. ✅ **提供兼容层** - 6 个月过渡期
5. ✅ **自动化迁移工具** - 降低用户迁移成本

### 破坏性变更

⚠️ **本次重构包含破坏性变更，但提供 6 个月兼容期**

- Import 路径变更（旧路径仍可用，但标记为 deprecated）
- 计划在 v2.0.0 完全移除兼容层

---

## 🗺️ 包重组计划

### 第一阶段：提升常用包到根目录

#### 1. captcha (验证码)

**当前位置**: `sdk/pkg/captcha`  
**新位置**: `captcha`  
**原因**: 高频使用的公共包，应在根目录

```go
// 旧导入 (v1.7.0 及之前)
import "github.com/go-admin-team/go-admin-core/sdk/pkg/captcha"

// 新导入 (v1.8.0+)
import "github.com/go-admin-team/go-admin-core/captcha"
```

**影响范围**:
- go-admin-pro: `app/admin/apis/sys_login.go`
- 预计外部使用者: 中等

#### 2. jwtauth (JWT 认证)

**当前位置**: `sdk/pkg/jwtauth`  
**新位置**: `jwtauth`  
**原因**: 核心认证功能，应在根目录

```go
// 旧导入
import "github.com/go-admin-team/go-admin-core/sdk/pkg/jwtauth"

// 新导入
import "github.com/go-admin-team/go-admin-core/jwtauth"
```

**影响范围**:
- go-admin-pro: `common/middleware/auth.go`, `common/middleware/handler.go`
- 预计外部使用者: 高

#### 3. response (响应封装)

**当前位置**: `sdk/pkg/response`  
**新位置**: `response`  
**原因**: API 响应工具，高频使用

```go
// 旧导入
import "github.com/go-admin-team/go-admin-core/sdk/pkg/response"

// 新导入
import "github.com/go-admin-team/go-admin-core/response"
```

**影响范围**:
- go-admin-pro: 所有 APIs 文件
- 预计外部使用者: 非常高

#### 4. casbin (权限管理)

**当前位置**: `sdk/pkg/casbin`  
**新位置**: `casbin`  
**原因**: 权限核心，应在根目录

```go
// 旧导入
import "github.com/go-admin-team/go-admin-core/sdk/pkg/casbin"

// 新导入
import "github.com/go-admin-team/go-admin-core/casbin"
```

**影响范围**:
- go-admin-pro: `common/middleware/permission.go`
- 预计外部使用者: 中等

---

### 第二阶段：简化包名

#### 5. observability → observe

**当前位置**: `observability/audit`  
**新位置**: `observe/audit`  
**原因**: 14 个字符太长，不便输入

```go
// 旧导入
import "github.com/go-admin-team/go-admin-core/observability/audit"

// 新导入
import "github.com/go-admin-team/go-admin-core/observe/audit"
```

**影响范围**: 内部使用，外部影响小

---

### 第三阶段：解决命名冲突

#### 6. sdk/config 冲突

**问题**: 与根目录 `config/` 冲突

**解决方案**: 
- 选项 A: 重命名为 `sdkconfig`
- 选项 B: 移入 `internal/sdk/config` (v2.0.0)

**当前决策**: 暂时保留，v2.0.0 移入 internal

#### 7. tools/gorm/logger 冲突

**问题**: 与根目录 `logger/` 冲突

**解决方案**: 重命名为 `gormlog`

```go
// 旧导入
import "github.com/go-admin-team/go-admin-core/tools/gorm/logger"

// 新导入
import "github.com/go-admin-team/go-admin-core/tools/gorm/gormlog"
```

## 🛡️ 兼容层设计

### 兼容层实现

在旧路径保留 `deprecated.go` 文件，重新导出新位置的 API：

```go
// sdk/pkg/captcha/deprecated.go
package captcha

import (
    newcaptcha "github.com/go-admin-team/go-admin-core/captcha"
)

// Deprecated: 使用 github.com/go-admin-team/go-admin-core/captcha 替代
// 本包将在 v2.0.0 移除
type Config = newcaptcha.Config

// Deprecated: 使用 github.com/go-admin-team/go-admin-core/captcha 替代
var New = newcaptcha.New

// Deprecated: 使用 github.com/go-admin-team/go-admin-core/captcha 替代
var Generate = newcaptcha.Generate

// ... 导出所有公共 API
```

### 兼容层特性

✅ **功能完整**: 所有 API 100% 可用  
✅ **编译告警**: 使用 `// Deprecated:` 标记  
✅ **IDE 提示**: 现代 IDE 会提示废弃  
✅ **过渡期**: 6 个月（至 v2.0.0）

## 🔧 迁移工具

### 自动迁移脚本

提供 `tools/migrate-v1.8.sh` 脚本：

```bash
#!/bin/bash
# 自动迁移 import 路径

# 使用方法
./tools/migrate-v1.8.sh /path/to/your/project
```

**功能**:
1. ✅ 自动替换 import 路径
2. ✅ 生成迁移报告
3. ✅ 支持预览模式（--dry-run）
4. ✅ 备份原文件

### 手动迁移

**步骤 1**: 查找旧导入

```bash
grep -r "sdk/pkg/captcha" .
grep -r "sdk/pkg/jwtauth" .
grep -r "sdk/pkg/response" .
grep -r "sdk/pkg/casbin" .
```

**步骤 2**: 替换导入路径

使用编辑器的全局替换功能：

```
查找: github.com/go-admin-team/go-admin-core/sdk/pkg/captcha
替换: github.com/go-admin-team/go-admin-core/captcha
```

**步骤 3**: 运行测试

```bash
go mod tidy
go test ./...
```
